# Day 25 – Linear Feedback Shift Register (LFSR)

## Overview

Today I learned about **Linear Feedback Shift Registers (LFSRs)**, a hardware-efficient technique used to generate pseudo-random sequences using shift registers and XOR-based feedback logic.

---

## What is an LFSR?

An LFSR is a shift register where the new input bit is generated by XORing selected register bits called **taps**.

### Working Principle

1. Data shifts by one position every clock cycle.
2. Selected tap bits are XORed.
3. The XOR result is fed back as the new input bit.

---

## Primitive vs Non-Primitive Polynomials

### Primitive Polynomial
- Produces a maximal-length sequence.
- Generates all non-zero states before repeating.
- Example: **x³ + x + 1**

### Non-Primitive Polynomial
- Produces shorter repeating sequences.
- Does not cover all possible states.
- Example: **x³ + 1**

---

## Tap Selection

- Taps determine which register bits participate in feedback generation.
- Proper tap selection is essential for achieving maximal-length sequences.

Example:

```text
Feedback = Tap1 ⊕ Tap2
```

---

## 4-Bit LFSR Example

Polynomial:

```text
x⁴ + x³ + 1
```

Tap Positions:

```text
[4,3]
```

Verilog Feedback Logic:

```verilog
assign feedback = lfsr_out[3] ^ lfsr_out[2];
lfsr_out <= {lfsr_out[2:0], feedback};
```

## Applications

- Pseudo-Random Number Generation (PRNG)
- CRC Generation
- Data Scrambling
- Communication Systems
- Built-In Self-Test (BIST)

---

## Key Takeaways

- LFSRs generate deterministic pseudo-random sequences.
- XOR feedback creates the next state.
- Primitive polynomials produce maximal-length sequences.
- All-zero state should be avoided.
- LFSRs provide efficient random sequence generation with minimal hardware.
